# Contributing to fabricXYZ

Thank you for your interest in contributing to fabricXYZ! This document provides guidelines for contributing to this Microsoft Fabric toolbox collection.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs or suggest features
- Clearly describe the issue including steps to reproduce
- Include relevant code snippets or screenshots if applicable

### Submitting Changes

1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes following the project structure
4. Commit your changes (`git commit -m 'Add some amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

## Project Structure

- `/accelerators` - Complete project templates and accelerators
- `/snippets` - Small reusable code snippets
- `/toolbox` - Utility scripts and tools
- `/docs` - Documentation and guides

## Contribution Guidelines

### For Accelerators

- Include a README.md explaining the purpose and usage
- Provide clear setup instructions
- Include example configurations
- Document any prerequisites

### For Snippets

- Keep snippets focused and single-purpose
- Add clear comments explaining the code
- Include usage examples
- Specify the Fabric environment (Lakehouse, Warehouse, etc.)

### For Toolbox Items

- Document all parameters and options
- Include error handling
- Provide usage examples
- Follow Python best practices if applicable

## Code Quality

- Write clear, readable code
- Add comments for complex logic
- Test your contributions
- Follow existing code style

## Documentation

- Update relevant documentation
- Add inline comments where necessary
- Include examples in your documentation

## Questions?

Feel free to open an issue for any questions about contributing!
